# 2026-06-25 작업 기록

## 1. 계획 및 목표
- **건물 컨텍스트 UI(선택 UI) 시스템 구축**: 건물을 좌클릭하여 철거, 업그레이드, 이동(재배치) 등을 할 수 있는 통합 팝업 패널을 구현.
- **건물 개별 레벨링 시스템 연동**: 타워 밸런스 확장과 생산 건물의 속도 향상을 위해 레벨(`level`) 개념을 도입하고 수치를 스케일링.
- **공급기 필터 UI 개선**: 기존에 공간을 차지하던 공급기 전용 투명 물리 버튼들을 제거하고, 건물 컨텍스트 UI의 드롭다운(OptionButton)으로 깔끔하게 편입.

## 2. 작업 처리 내역 (Execution)
- `player.gd`에 `building_context_panel` 추가 및 UI 디자인 (이름, 레벨 표시, 버튼들 배치).
- 몬스터 코어를 소모하여 업그레이드를 수행하고, 건물의 `set_meta("level", value)` 업데이트.
- 타워 스크립트(`turret.gd`, `sniper_turret.gd`, `shotgun_turret.gd`, `laser_turret.gd`, `missile_turret.gd`)와 기능성 건물(`drill.gd`, `processor.gd`)에서 `get_meta("level")`을 읽어와 데미지 및 쿨타임을 동적으로 조절하도록 수정.
- 공급기(`provider.gd`) 내부의 복잡한 버튼/레이블 코드 삭제 및 메타데이터(`filter_idx`)만 업데이트하도록 경량화.

## 3. 오류 발생 및 트러블슈팅 (Errors & Fixes)

### 오류 1: 파서 에러 (Parser Error)
- **문제**: `player.gd`의 `handle_building` 함수 내에서 `local_pos` 변수가 이미 선언되어 중복 선언(Parser Error) 발생.
- **해결**: 중괄호 스코프 내에서 실수로 `var local_pos = ...`가 두 번 호출된 부분을 찾아 뒤의 `var` 키워드를 삭제하여 해결함.

### 오류 2: UI 점핑 (업그레이드 시 패널 이동)
- **문제**: 건물을 업그레이드(클릭)할 때마다 UI가 리프레시되면서, 팝업 패널이 현재 마우스 커서 위치(버튼 위)로 계속 따라다니며 도망가는 현상 발생.
- **해결**: `open_context_ui(building, refresh = false)` 형태로 파라미터를 분리하고, 단순 갱신(`refresh = true`) 시에는 패널의 좌표 이동 로직을 우회하도록 처리함.

### 오류 3: 클릭 감지 어긋남 (히트박스 문제)
- **문제**: 건물의 시각적 중앙(노란색 사각형 등)을 누르면 클릭이 인식되지 않고, 상단 끝/가장자리 가장자리에 걸친 부분을 눌러야만 UI 창이 뜨는 현상.
- **원인 분석**: 
  1. 처음에 마우스 좌표 변환(`get_global_mouse_position()` vs `event.position`) 오차인 줄 알았으나, 가장 근본적인 원인은 UI 노드의 마우스 필터.
  2. 모든 건물 스크립트가 임시 스프라이트로 `ColorRect.new()`를 생성해서 쓰고 있었는데, Godot 4에서 `Control`을 상속하는 `ColorRect`는 기본적으로 `mouse_filter = Control.MOUSE_FILTER_STOP`을 가짐.
  3. 따라서 건물의 시각적 사각형을 누르면 이벤트가 소비(Consume)되어 버려 `player.gd`의 `_unhandled_input`이 호출되지 않았음. (빈 공간을 누를 때만 호출됨)
- **해결**: 프로젝트 내의 모든 건물 스크립트(`shotgun_turret`, `processor`, `drill` 등)와 `player.gd`의 미리보기(`preview_rect`)를 스캔하여, `ColorRect` 생성 직후 **`mouse_filter = Control.MOUSE_FILTER_IGNORE` (값: 2)** 를 적용하도록 전면 패치 완료.
